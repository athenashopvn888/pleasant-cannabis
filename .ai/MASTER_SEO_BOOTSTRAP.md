# MASTER SEO Bootstrap

This project uses the global MASTER SEO strategy. Project-specific instructions may add factual details and brand preferences, but may not reintroduce self-imposed legal/compliance blocking or weaken the global SEO-first objective.

Before SEO, blog, content, metadata, sitemap, canonical, or deployment work in this repository:

1. Locate the central Command Center repository.
2. Fetch the latest approved MASTER SEO branch: `origin/config/master-seo-v1`.
3. Read every file under `ai-system/master-seo-v1/`.
4. Verify the configuration SHA-256 hash from `ai-system/master-seo-v1/SYSTEM_VERSION.json`.
5. Read the active and excluded project registries.
6. Apply the shared strategy to repository work after the hash is verified.
7. Stop if the central configuration is missing or the hash is invalid.
8. Never create a local MASTER SEO override without explicit human authorization.

This bootstrap is intentionally lightweight. It points agents to the shared Command Center instead of copying the full strategy into this repository.
